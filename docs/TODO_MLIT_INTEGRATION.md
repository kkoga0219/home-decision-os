## 1. エリア平均㎡単価の実データ化 [高優先]

**現状**: `area_stats.py` にハードコードした概算値（仮データ）
**MLIT対応**: XIT001で尼崎市・西宮市等の中古マンション取引を取得し、
駅ごとの㎡単価中央値・平均値を算出する。

```
GET /ex-api/external/XIT001
  area=28 (兵庫県), city=28202 (尼崎市)
  from=20231, to=20254
→ TradePrice, Area, NearestStation, BuildingYear 等
→ マンション取引のみフィルタ → 駅別に集計
```

**影響範囲**: `area_stats.py`, `enrichment.py`, `rent_estimator.py` (Tier2)

## 8. ML回帰モデルによる価格・賃料推定 [中優先]

**現状**: ルールベースの3層推定（SUUMO実データ → ブレンド → 利回り逆算）
**目標**: MLIT取引データ + SUUMOデータを教師データとし、機械学習による回帰モデルを構築。

**特徴量（カテゴリカル含む）**:
  - 数値: 面積(㎡), 築年数, 駅徒歩(分), 階数, 総戸数
  - カテゴリカル: 用途地域, 間取りタイプ(1K/2LDK等), 構造(RC/SRC等),
    最寄り駅, 市区町村, ハザード区分
  - MLIT由来: 取引価格, 取引件数(流動性指標), 四半期トレンド

**モデル候補**:
  - LightGBM / XGBoost: カテゴリカル特徴量を直接扱える、解釈性も高い
  - Ridge/Lasso回帰: ベースラインモデルとして
  - 将来的にNN (embedding層でカテゴリカルを扱う)

**実装ステップ**:
  1. MLIT APIから取引データを定期取得しDBに蓄積（→ TODO #1）
  2. SUUMO賃料データと結合して教師データセット構築
  3. 特徴量エンジニアリング（築年数→経年、駅距離→バケット等）
  4. LightGBMで㎡単価予測モデル / 賃料予測モデルを学習
  5. `rent_estimator.py` のTier1として組み込み（信頼度: VERY_HIGH）
  6. 定期再学習パイプライン（月次 or 四半期）

**評価指標**: MAPE (Mean Absolute Percentage Error) 目標 < 10%

**影響範囲**: `rent_estimator.py`, `area_stats.py`, 新規 `ml_models/` ディレクトリ

## MLIT APIでは改善できないもの

| 項目 | 理由 |
|------|------|
| 固定資産税評価額 | 市区町村の課税データ。公開APIなし |
| 実際の賃料データ | MLITは売買取引のみ。賃料はSUUMO等に依存 |
| マンション管理状態 | 個別マンションの管理組合情報は非公開 |
| 修繕積立金の適正値 | 国交省ガイドライン値は別途参照可能だが、APIではない |
